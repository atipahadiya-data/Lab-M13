## Part 1 — Fix 5 Broken DAG Configurations

### Broken Config 1: The DAG That Never Stops Changing

**Diagnosis:**
- `start_date=datetime.now()` changes every time the DAG file is parsed
- Airflow scheduler parses DAG files frequently → start_date keeps moving
- DAG may never trigger or appear unstable

**Fix:**
```python
with DAG(
    dag_id="daily_sales_report",
    start_date=datetime(2024, 1, 1),  # static start_date in the past
    schedule_interval="@daily",
    catchup=False,
) as dag:
    generate_report = PythonOperator(
        task_id="generate_report",
        python_callable=generate_sales_report,
    )
```
**Alternative fix:** keep catchup=True but limit concurrent runs with max_active_runs=3.

# Broken Config 3: The Non-Deterministic Task

**Diagnosis:**

- Task uses datetime.now() → non-deterministic results
- Breaks backfills and reproducibility

Fix:
```python
def process_daily_data(**context):
    target_date = context["ds"]  # execution_date as YYYY-MM-DD
    query = f"SELECT * FROM orders WHERE order_date = '{target_date}'"
    print(f"Processing data for: {target_date}")

with DAG(
    dag_id="daily_order_processing",
    start_date=datetime(2024, 1, 1),
    schedule_interval="@daily",
    catchup=False,
) as dag:
    process = PythonOperator(
        task_id="process_daily_data",
        python_callable=process_daily_data,
        provide_context=True,
    )
```

# Broken Config 5: The Template-Unaware Midnight Task
**Diagnosis:**
Uses datetime.today() - timedelta(days=1) → wall-clock dependent
Breaks idempotency and backfills

**Fix:**
```python

def process_yesterdays_data(**context):
    target_date = context["ds"]
    file_path = f"/data/raw/{target_date}/events.csv"
    print(f"Processing file: {file_path}")

with DAG(
    dag_id="daily_event_processing",
    start_date=datetime(2024, 1, 1),
    schedule_interval="0 0 * * *",
    catchup=False,
) as dag:
    process = PythonOperator(
        task_id="process_yesterdays_data",
        python_callable=process_yesterdays_data,
        provide_context=True,
    )
```
**Alternative using Jinja templates:**
- BashOperator: bash_command="python process.py --date {{ ds }}"
- File path: /data/raw/{{ ds }}/events.csv
- No-dash: /data/raw/{{ ds_nodash }}/events.csv

# Scheduling Cheat Sheet
| Concept               | Wrong Way                           | Right Way                 |
| --------------------- | ----------------------------------- | ------------------------- |
| start_date            | datetime.now()                      | datetime(2024, 1, 1)      |
| Target date in task   | datetime.today()                    | context["ds"] or {{ ds }} |
| File date (no dashes) | datetime.today().strftime('%Y%m%d') | {{ ds_nodash }}           |
| Data range            | Manual timedelta arithmetic         | data_interval_start/end   |
| Avoid mass backfill   | catchup=True (default)              | catchup=False             |

# Key Rules:

- start_date must be STATIC
- execution_date = start of data interval
- First run triggers at END of first interval
- Tasks must be DETERMINISTIC — use templates, not wall-clock
- catchup=False unless intentional backfill is needed
