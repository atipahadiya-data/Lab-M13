## Scenario 1: Linear DAG – E-Commerce Daily Pipeline

### Task List

| Task ID               | Description                                    | Operator Type     |
|-----------------------|------------------------------------------------|-----------------|
| extract_orders        | Pull today's orders from transactional DB     | PythonOperator  |
| clean_orders          | Remove duplicates, fix nulls, standardize     | PythonOperator  |
| enrich_with_customers | Join orders with customer dimension table     | PythonOperator  |
| load_to_warehouse     | Insert enriched data into analytics warehouse | PythonOperator  |
| send_report_email     | Email daily order summary to stakeholders     | EmailOperator   |

### Dependency Diagram (Text-Based)
---
[extract_orders] --> [clean_orders] --> [enrich_with_customers] --> [load_to_warehouse] --> [send_report_email]

---

### Airflow-Style Pseudocode
```python
from airflow import DAG
from airflow.operators.python import PythonOperator
from airflow.operators.email import EmailOperator
from datetime import datetime

with DAG(
    dag_id="ecommerce_daily_pipeline",
    start_date=datetime(2024, 1, 1),
    schedule_interval="@daily",
    catchup=False,
) as dag:

    extract_orders        = PythonOperator(task_id="extract_orders", python_callable=extract_orders_fn)
    clean_orders          = PythonOperator(task_id="clean_orders", python_callable=clean_orders_fn)
    enrich_with_customers = PythonOperator(task_id="enrich_with_customers", python_callable=enrich_fn)
    load_to_warehouse     = PythonOperator(task_id="load_to_warehouse", python_callable=load_fn)
    send_report_email     = EmailOperator(task_id="send_report_email", to="team@company.com")

    # Linear dependencies
    extract_orders >> clean_orders >> enrich_with_customers >> load_to_warehouse >> send_report_email
```

# Failure Propagation Analysis
- extract_orders fails: downstream tasks skipped, no data loaded, report not sent.
- load_to_warehouse fails: only send_report_email skipped, earlier steps succeed.
- send_report_email fails: pipeline otherwise successful, data loaded.
Parallelism
None; strictly sequential.

Task List

| extract_billing   | Pull invoice records from Stripe API      | PythonOperator |
| extract_support   | Pull support tickets from Zendesk API     | PythonOperator |
| merge_data        | Join all three datasets on customer_id    | PythonOperator |
| aggregate_metrics | Compute customer-level KPIs               | PythonOperator |
| load_to_warehouse | Insert aggregated data into warehouse     | PythonOperator |

# Dependency Diagram (Text-Based)

                     \

                     /

# Airflow-Style Pseudocode
```python
from airflow import DAG
from airflow.operators.python import PythonOperator
from datetime import datetime

with DAG(
    dag_id="multi_source_analytics",
    start_date=datetime(2024, 1, 1),
    schedule_interval="@daily",
    catchup=False,
) as dag:

    extract_crm     = PythonOperator(task_id="extract_crm", python_callable=extract_crm_fn)
    extract_billing = PythonOperator(task_id="extract_billing", python_callable=extract_billing_fn)
    extract_support = PythonOperator(task_id="extract_support", python_callable=extract_support_fn)
    merge_data      = PythonOperator(task_id="merge_data", python_callable=merge_fn)
    aggregate       = PythonOperator(task_id="aggregate_metrics", python_callable=aggregate_fn)
    load_warehouse  = PythonOperator(task_id="load_to_warehouse", python_callable=load_fn)

    # Parallel extractions fan-in
    [extract_crm, extract_billing, extract_support] >> merge_data >> aggregate >> load_warehouse
```
# Failure Propagation Analysis
- extract_billing fails: merge_data skipped, aggregate & load skipped.
- merge_data fails: upstream extractions succeed, downstream skipped.
- slow extraction: other parallel extractions unaffected; merge waits.
# Parallelism
- extract_crm, extract_billing, extract_support run concurrently.
- merge_data → aggregate → load sequential.

# Scenario 3: Conditional Branching DAG – Data Quality Pipeline
Task List

| Task ID           | Description                                    | Operator Type        |
| ----------------- | ---------------------------------------------- | -------------------- |
| extract_data      | Pull raw data from source system               | PythonOperator       |
| validate_data     | Run quality checks (nulls, schema, thresholds) | PythonOperator       |
| branch_on_quality | Choose path based on validation result         | BranchPythonOperator |
| transform_data    | Apply transformations (valid path)             | PythonOperator       |
| load_to_warehouse | Insert into warehouse (valid path)             | PythonOperator       |
| quarantine_data   | Move bad data to quarantine (invalid path)     | PythonOperator       |
| send_alert        | Notify team of quality failure (invalid path)  | EmailOperator        |

# Dependency Diagram (Text-Based)
                                          ┌──(valid)──> [transform_data] --> [load_to_warehouse]
                                          │
[extract_data] --> [validate_data] --> [branch_on_quality]
                                          │
                                          └──(invalid)-> [quarantine_data] --> [send_alert]

# Airflow-Style Pseudocode
```python 

from airflow import DAG
from airflow.operators.python import PythonOperator, BranchPythonOperator
from airflow.operators.email import EmailOperator
from datetime import datetime

def decide_branch(**context):
    validation_result = context["ti"].xcom_pull(task_ids="validate_data")
    if validation_result == "passed":
        return "transform_data"
    else:
        return "quarantine_data"

with DAG(
    dag_id="data_quality_pipeline",
    start_date=datetime(2024, 1, 1),
    schedule_interval="@daily",
    catchup=False,
) as dag:

    extract_data       = PythonOperator(task_id="extract_data", python_callable=extract_fn)
    validate_data      = PythonOperator(task_id="validate_data", python_callable=validate_fn)
    branch_on_quality  = BranchPythonOperator(task_id="branch_on_quality", python_callable=decide_branch)
    transform_data     = PythonOperator(task_id="transform_data", python_callable=transform_fn)
    load_to_warehouse  = PythonOperator(task_id="load_to_warehouse", python_callable=load_fn)
    quarantine_data    = PythonOperator(task_id="quarantine_data", python_callable=quarantine_fn)
    send_alert         = EmailOperator(task_id="send_alert", to="data-team@company.com")

    # Linear lead-in
    extract_data >> validate_data >> branch_on_quality

    # Branching paths
    branch_on_quality >> transform_data >> load_to_warehouse
    branch_on_quality >> quarantine_data >> send_alert
```
# Failure Propagation Analysis
- extract_data fails: downstream tasks skipped.
- validate_data fails: branch skipped → no load or quarantine.
- quarantine_data fails: send_alert skipped → team not notified.
- data valid: valid path executes; invalid path skipped.
# Parallelism
- Limited; branch paths mutually exclusive.
- Could add parallelism if multiple independent validations run before branch.
